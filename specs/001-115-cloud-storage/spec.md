# Feature Specification: 115 Cloud 存储库在线播放

**Feature Branch**: `[001-115-cloud-storage]`  
**Created**: 2026-01-31  
**Status**: Draft  
**Input**: User description: "参考openlist项目115 cloud源码以及当前项目115open存储库逻辑，为当前项目添加115 cloud存储库集成"

## Assumptions

- 用户拥有可用的 115 账号，并同意在应用内完成授权后访问其个人云盘内容。
- 首期范围聚焦“浏览云盘文件 + 选择视频直接播放”，不包含上传、删除、移动、分享等文件管理能力。
- 仅覆盖个人云盘使用场景；企业/团队/共享空间等形态不在首期范围内。
- 115 Cloud 作为“存储源”的新增、管理、文件浏览、播放入口与交互路径与应用内现有存储源保持一致（与“百度网盘”“115 Open”的浏览/播放体验对齐），除授权页外不引入独立的专用操作路径。
- 授权完成后，在授权仍有效的前提下，用户关闭并重新打开应用无需重复授权即可继续浏览与播放；仅当授权失效或用户主动解绑时才需要重新授权。
- 授权相关信息属于敏感数据：在 UI 与日志中不得输出完整敏感信息（仅允许遮罩/截断），移除存储源时必须清理对应的本地授权数据。
- 该能力依赖 115 Cloud 服务可用性与授权策略；如服务端策略变化可能影响可用性与体验。

## Clarifications

### Session 2026-01-31

- Q: 115 Cloud 的授权方式采用哪种用户交互？ → A: 默认采用“应用内展示二维码 → 用户使用 115 客户端扫码确认”的授权流程；不要求输入账号/密码。
- Q: 新增 115 Cloud 存储源后，默认可浏览的范围是？ → A: 从云盘根目录开始，全盘可浏览（在用户自身权限允许范围内）。
- Q: 授权失效时的恢复策略是？ → A: 阻止继续访问并提示用户重新授权；重新授权成功后可继续浏览与播放。

### Session 2026-02-01

- Q: 同一账号的重复添加规则是？ → A: 同一类型存储库内不允许重复添加相同账号；但 115 Open 与 115 Cloud 视为不同类型，允许分别添加同一账号各 1 个。

## Out of Scope

- 账号密码登录、短信验证码登录等非扫码授权方式。
- 通过“模拟网页版/抓包”等方式导入浏览器登录态/会话信息的高级玩法（如后续需要，另立需求）。
- 云盘文件的上传、删除、重命名、移动、复制、分享链接创建/管理。
- 115 侧离线下载/转存任务管理。
- 将网盘视频“完整下载到本地”作为首要播放路径（可作为后续增强）。
- 企业/团队/共享空间等非个人云盘形态。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 挂载 115 Cloud 并播放视频 (Priority: P1)

用户在“存储/文件浏览”中新增一个“115 Cloud”存储源，通过“二维码扫码授权”完成连接后可以浏览云盘目录，选择一个视频文件后直接进入播放器并开始播放。

**Why this priority**: 这是“把 115 Cloud 当作存储源来播放视频”的核心闭环；没有它就无法达成主要价值。  
**Independent Test**: 使用一个包含至少 1 个目录与 1 个视频文件的 115 账号：完成添加存储源 → 浏览到视频 → 点击播放，验证能进入播放器并成功开始播放。

**Acceptance Scenarios**:

1. **Given** 用户未添加过 115 Cloud 存储源且网络可用，**When** 用户选择“新增存储源 → 115 Cloud”，**Then** 系统展示可扫码二维码并引导用户完成授权。
2. **Given** 用户完成扫码授权，**When** 系统创建 115 Cloud 存储源，**Then** 用户能看到云盘根目录列表，并可进入子目录后再返回继续浏览。
3. **Given** 用户在文件列表中选择一个受支持的视频文件，**When** 用户点击播放，**Then** 系统进入播放器并开始播放（出现画面或听到声音）。
4. **Given** 授权失效或网络不可用，**When** 用户尝试浏览或播放，**Then** 系统给出清晰提示，并提供“重试/重新授权/返回”等可执行操作入口。
5. **Given** 用户已添加过某个 115 账号的 115 Cloud 存储源，**When** 用户尝试再次添加同一账号的 115 Cloud 存储源，**Then** 系统拦截并提示用户改为更新既有存储源（该规则不影响用户在其他类型存储库中添加同一账号）。

### Functional Requirements

- **FR-001**: 系统必须在“新增存储源”入口提供“115 Cloud”选项，并在添加成功后在存储源列表中可见。
- **FR-002**: 系统必须提供“二维码扫码授权”流程完成 115 Cloud 账号绑定，不要求用户输入账号/密码；用户可取消、刷新二维码、重试，且每种结果均有明确提示。
- **FR-003**: 授权成功后，系统必须允许用户从云盘根目录开始浏览文件列表，支持进入目录与返回上级目录继续浏览。
- **FR-004**: 系统必须在文件列表中区分“目录/文件”，并在可展示时提供基础信息（如名称、大小、修改时间）。
- **FR-005**: 系统必须允许用户对受支持的视频文件发起在线播放，并进入播放器开始播放。
- **FR-006**: 系统必须阻止对不受支持文件类型的播放尝试，并向用户说明原因与可选操作（返回/更换文件）。
- **FR-007**: 系统必须在断网/弱网、文件不存在、授权失效等异常场景下给出清晰提示，并提供可执行的恢复入口（重试/重新授权/返回）。
- **FR-008**: 系统必须支持用户移除已添加的 115 Cloud 存储源；移除后该存储源不可再访问，且对应的本地授权数据必须被清理。
- **FR-009**: 系统必须阻止在 115 Cloud 存储源中重复添加同一 115 账号；但该限制仅对 115 Cloud 生效，不应影响用户在其他类型存储库（如 115 Open）中添加同一账号。
- **FR-010**: 系统必须使 115 Cloud 存储源在“新增、选择、管理、文件浏览、播放”等环节的入口与交互与其他存储源保持一致（除扫码授权页外）。
- **FR-011**: 系统必须提供与现有存储源一致的文件浏览定位能力：支持刷新当前目录列表、按应用现有排序选项排序，并支持在当前目录递归范围内按关键字搜索内容（仅返回匹配的可播放视频文件）。
- **FR-012**: 系统必须对授权相关敏感信息进行保护：在 UI 与日志中不得输出完整敏感信息（仅允许遮罩/截断），错误提示也不得泄露完整敏感信息。
- **FR-013**: 系统必须在授权仍有效的前提下支持“重启后免重复授权”：用户关闭并重新打开应用仍可直接浏览与播放；当授权失效时才提示重新授权。
- **FR-014**: 系统必须保证用户从 115 Cloud 发起的播放在应用当前支持的所有播放方式下均可启动；若某播放方式启动失败，应给出明确提示并允许用户切换到其他播放方式重试。

### Acceptance Criteria (by requirement)

- **FR-001**: 在新增存储源入口中可选择“115 Cloud”，且新增成功后在存储列表中可见。
- **FR-002**: 授权流程展示可扫码二维码且不出现账号/密码输入项；用户可取消、刷新二维码、重试；授权成功/失败/取消均有明确反馈。
- **FR-003**: 授权成功后默认可看到云盘根目录文件列表；进入与返回目录后列表内容正确且可继续操作。
- **FR-004**: 列表中可区分文件与目录；在可展示时可查看文件大小与修改时间等基础信息。
- **FR-005**: 点击受支持的视频文件可进入播放器并开始播放；播放开始的判定为“出现画面或听到声音”。
- **FR-006**: 点击不受支持文件会被拦截并提示原因；用户可返回继续浏览或选择其他文件。
- **FR-007**: 断网/弱网、文件不存在、授权失效等场景下，用户能看到明确提示与可执行的恢复操作入口；恢复失败不会导致应用崩溃。
- **FR-008**: 用户移除存储源后，该存储源从列表消失；再次进入不再显示该账号内容且需要重新授权才可恢复。
- **FR-009**: 在 115 Cloud 中对同一账号重复添加会被拦截并提示用户改为更新既有存储源；但用户仍可在其他类型存储库（如 115 Open）中添加同一账号且不受影响。
- **FR-010**: 115 Cloud 与其他存储源共用同一套入口与页面结构：新增入口在同一处、存储源列表呈现方式一致、文件浏览页面结构一致、播放入口一致、管理/移除入口一致；除扫码授权页外不出现“仅 115 Cloud 可见”的独立流程。
- **FR-011**: 在 115 Cloud 文件列表页可执行刷新/排序/搜索；排序项与现有存储源一致；搜索可在当前目录递归范围内返回匹配的可播放视频文件，且清空搜索后恢复当前目录列表与上下文。
- **FR-012**: 配置页默认不展示完整敏感信息（可采用遮罩/隐藏等方式），且在错误提示与日志中不输出完整敏感信息；移除存储源后不再保留对应授权数据。
- **FR-013**: 在授权有效的前提下，关闭并重新打开应用后仍可直接浏览与播放；授权失效时访问会被阻止并提示重新授权；重新授权成功后可继续浏览与播放。
- **FR-014**: 在设置中分别切换不同播放方式：点击同一个受支持的视频文件均可进入播放器并尝试开始播放；若某播放方式启动失败，应给出明确提示并允许用户切换到其他播放方式重试。

### Key Entities *(include if feature involves data)*

- **115 Cloud 存储源**: 代表一个已添加的 115 Cloud 连接（展示名、账号标识、授权状态、最近一次成功访问时间）；同一账号在 115 Cloud 中应唯一，但不与其他类型存储库的账号添加规则冲突。
- **扫码授权会话**: 代表一次二维码授权流程（二维码、有效期、当前状态、最终结果）。
- **云盘文件项**: 代表一个目录或文件（名称、类型、路径/层级位置、可选的大小与修改时间）。
- **播放请求**: 代表一次从云盘文件启动播放的行为（目标文件、发起时间、结果状态、失败原因）。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 新用户从“新增存储源”开始，到完成扫码授权并看到云盘文件列表的流程，90% 的情况下可在 2 分钟内完成。
- **SC-002**: 在网络可用且授权有效的前提下，用户打开已添加的 115 Cloud 存储源后，90% 的情况下可在 3 秒内看到首屏文件列表。
- **SC-003**: 用户点击一个受支持的视频文件后，80% 的情况下可在 5 秒内看到播放器开始播放（出现画面或听到声音）。
- **SC-004**: 在授权有效且网络可用的前提下，用户首次尝试从 115 Cloud 播放受支持视频的成功率达到 90% 以上。
- **SC-005**: 当授权失效或网络不可用时，用户能够通过提示完成“重试/重新授权/返回”的恢复操作，且无需强退应用即可继续使用其他功能（崩溃率 ≤ 0.5%）。
