# 观察项（Out of Build Scope）清单

本清单用于记录：**不在 `settings.gradle.kts` 的 `include(...)` 范围内**，但在仓库中存在的目录/文件（历史遗留、第三方拷贝、误解包产物、仅开发期工具等）。

## 1. 策略（强制）

- 观察项不阻塞 US1/US2 交付，但需要“可追溯”：
  - 路径
  - 类型（历史/第三方/误产物/工具/空目录等）
  - 风险/影响（为什么需要记录）
  - 建议动作（保留/清理/迁移/补充说明）
- 若观察项影响到主线排查判断（例如重复实现证据来自观察项），需要在对应模块报告里明确标注其“非构建范围来源”。
- 若观察项需要纳入治理（例如清理、迁移、或重新纳入构建），在 US3 的全局汇总里输出对应 Task。

## 2. 当前清单（初始）

> 说明：以下为基于仓库现状的初始盘点，后续可增量维护。

| 路径 | 类型 | 现状/风险 | 建议动作 |
|---|---|---|---|
| `other/` | 工具/第三方 | 包含 jar 与脚本（非构建模块），容易被误当成业务依赖来源 | 明确用途（README 或注释），若仅历史工具可迁移到 `scripts/` 或归档 |
| `change/` | 历史记录 | 版本变更文本（不影响构建） | 保留（文档类），无需纳入模块排查 |
| `res/` | 误解包产物 | 仓库根目录 `res/` 已被 `.gitignore` 忽略（疑似 AAR 解包残留） | 若确认无用途，建议清理；避免与模块内 `src/main/res` 混淆 |
| `classes.jar` | 误解包产物 | 仓库根目录存在 `classes.jar`，已被 `.gitignore` 忽略 | 若确认无用途，建议清理；避免误提交/误引用 |
| `arm64-v8a/` `armeabi/` `armeabi-v7a/` | 空目录/遗留 | 根目录下存在空 ABI 目录，易造成误判 | 若长期为空且无脚本依赖，建议删除 |
| `jni/` `lib/` `kotlin/` `values/` | 空目录/遗留 | 根目录下存在空目录（与模块内同名语义可能冲突） | 若确认未被构建/脚本引用，建议删除或迁移到明确用途的目录 |
| `jdk-17.0.17+10/` | 工具链 | 仓库内置 JDK（非业务代码） | 保留（若用于构建一致性），无需纳入模块排查 |
| `.tmp/` | 临时目录 | 临时文件目录（已在 `.gitignore` 忽略） | 保留忽略规则；排查时不纳入证据来源 |
